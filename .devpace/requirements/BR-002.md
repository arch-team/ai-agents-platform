# BR-2: 对话执行

> 优先级：P0 | 关联：OBJ-1 | PF：3 个

## 用户故事

- 作为业务用户，我希望能与 AI Agent 自然对话，以便快速获取信息和完成任务

## 关键流程

```
消息发送 → Agent 处理 → 流式返回 → 历史管理
```

- **消息发送**：用户输入消息，发送到后端
- **Agent 处理**：Agent 接收消息，调用 LLM 和工具处理
- **流式返回**：SSE 流式返回 Agent 响应内容
- **历史管理**：保存对话历史，支持上下文延续

## 验收标准

- [ ] 用户能发送消息并收到 Agent 回复
- [ ] Agent 响应支持 SSE 流式输出，实时显示生成内容
- [ ] 对话历史正确保存，刷新页面后能恢复

## 异常场景

| 场景 | 处理方式 |
|------|---------|
| Agent 不可用 | Agent 处于停用或删除状态，提示用户选择其他 Agent |
| LLM 调用失败 | Bedrock API 超时或配额不足，显示错误并支持重试 |
| 工具调用失败 | 工具不可用或执行错误，Agent 需要优雅降级 |
| 消息太长 | 输入超出 Token 限制，提示用户缩短消息或分批发送 |

## 非功能需求

- **性能**：首条响应 < 3s，SSE 延迟 < 500ms
- **安全**：对话隔离，用户只能看到自己的对话
- **可用性**：流式连接异常断开时自动重连

## 产品功能

- PF-04: Agent 对话（消息收发 + SSE 流式）
- PF-05: 团队协作执行（多 Agent 协同）
- PF-06: Chat 对话页面
