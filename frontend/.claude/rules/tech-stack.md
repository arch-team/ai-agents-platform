## 1. 版本升级策略

### 升级分类

| 分类 | 范围 | 流程 |
|------|------|------|
| **主版本** | React, TypeScript, Vite | 团队评审 + 完整测试 |
| **次版本** | 状态管理、测试工具 | 自主升级 + 回归测试 |
| **补丁版本** | 所有库 | 自主升级 |

### 升级检查清单

- [ ] 阅读官方迁移指南
- [ ] 运行完整测试套件
- [ ] 检查类型兼容性
- [ ] 验证生产构建

---

## 2. 技术选型原则

### 选型标准

| 标准 | 权重 | 说明 |
|------|------|------|
| **生态成熟度** | 高 | 活跃社区、稳定维护 |
| **类型支持** | 高 | 完善的 TypeScript 支持 |
| **Bundle 大小** | 中 | 合理的包体积 |
| **学习曲线** | 中 | 团队易于上手 |

### 禁止使用

| 技术 | 原因 | 替代方案 |
|------|------|---------|
| Redux | 过于复杂 | Zustand |
| MobX | 非 React 惯用模式 | Zustand |
| lodash (全量) | Bundle 膨胀 | lodash-es (按需) |
