# 会话工作流规则 (Session Workflow)

> Claude Code 每次会话自动执行的三步循环协议

---

## §0 速查卡片

```
开始 → 读取 progress.md → 识别任务/依赖/遗留 → 向用户汇报
执行 → 加载参考规范 → 检查前置依赖 → TDD 实现 → 质量检查
结束 → 更新 progress.md 五个区域
```

### progress.md 更新清单

| 区域 | 更新内容 | 何时更新 |
|------|---------|---------|
| 当前状态 | "下一步"指向下一个待开始任务 | 每次会话结束 |
| 模块状态 | 模块整体完成时更新状态 | 模块完成时 |
| 任务拆解表 | 已完成任务状态改为"已完成"，填写会话日期 | 任务完成时 |
| 遗留事项 | 新增未完成项 / 清除已解决项 | 有变更时 |
| 上次会话 | 覆盖更新为本次会话记录（日期、完成、决策） | 每次会话结束 |

---

## §1 会话开始协议

### 必须执行

1. **读取** `docs/progress.md`
2. **识别**以下信息：
   - 当前 Milestone 和阶段
   - 待做任务列表（状态为"待开始"或"进行中"）
   - 遗留事项（上次会话未完成的问题）
   - 前置依赖关系
3. **向用户汇报**当前状态摘要

### 任务选择规则

- 如用户指定了任务编号 → 执行指定任务
- 如用户未指定任务 → 选择**编号最小的"待开始"任务**
- 如有"进行中"任务 → 优先继续该任务
- 如有遗留事项 → 提醒用户并建议优先处理

### 前置依赖检查

- 读取任务表"依赖"列
- 如前置任务未完成 → 提示用户，建议先完成前置任务
- 如前置任务被阻塞 → 报告阻塞原因，请用户决策

---

## §2 任务执行协议

### 执行前

1. 读取任务表中"参考规范"列指定的 rules 文件
2. 确认前置依赖已满足
3. 确认任务范围和交付物

### 执行中

- 编码任务遵循 TDD 工作流：先写测试 → 再写实现 → 运行测试验证
- 将任务状态更新为"进行中"

### 执行后 — 质量检查

编码任务完成后，自动运行以下检查：

```bash
ruff check backend/
mypy backend/
pytest backend/tests/
```

- 全部通过 → 任务标记为"已完成"
- 有失败项 → 修复后重新运行，直到通过

### ADR 触发条件

以下情况需要创建架构决策记录 (`docs/adr/00X-*.md`)：

- 面临两个以上可行技术方案需要选择
- 实际开发与现有规范产生冲突
- 引入新的外部依赖或第三方库
- 偏离 rules 中既定模式的设计决策

---

## §3 会话结束协议

### 必须更新 `docs/progress.md` 的五个区域

1. **当前状态**
   - 更新"下一步"为下一个待开始任务的描述
   - 如 Milestone 完成，更新里程碑状态

2. **模块状态**
   - 模块内所有任务完成 → 状态改为"已完成"
   - 模块有任务进行中 → 状态改为"进行中"

3. **任务拆解表**
   - 已完成任务：状态改为"已完成"，会话列填写当前日期
   - 部分完成任务：状态改为"进行中"，细节记入遗留事项

4. **遗留事项**
   - 新增：本次未完成的工作、发现的问题、跨会话待处理项
   - 清除：本次已解决的遗留项

5. **上次会话**（覆盖更新）
   - 日期：当前日期
   - 完成：本次完成的具体工作
   - 决策：本次做出的技术决策（如有）

---

## §4 Milestone 验收协议

当 Milestone 所有实现任务完成后，执行验收：

### 自动化检查

```bash
ruff check backend/
mypy --strict backend/
pytest --cov-fail-under=85 backend/tests/
```

### 规范对照

1. 对照 `backend/.claude/rules/checklist.md` 逐项检查
2. 对照 `docs/strategy/roadmap.md` 中对应 Milestone 的验收标准

### 验收结果处理

- 全部通过 → 更新 Milestone 状态为"已完成"，开始拆解下一个 Milestone
- 有未通过项 → 记录到遗留事项，创建修复任务

## §5 反馈循环

开发过程中各类产出的归档位置：

| 产出类型 | 归档位置 | 触发条件 |
|---------|---------|---------|
| 任务进度 | `docs/progress.md` | 每次会话结束 |
| 架构决策 | `docs/adr/00X-*.md` | 见 §2 ADR 触发条件 |
| 规范修正 | `backend/.claude/rules/*.md` | 发现规范与实际不符 |
| 遗留事项 | `docs/progress.md` 遗留事项区 | 任务未完成或发现跨会话问题 |
