### Step 1：打开平台，点击「创建 Agent」

1. 在浏览器中打开培训平台地址：`http://ai-agents-dev-546356512.us-east-1.elb.amazonaws.com`
2. 用公司账号登录（初次登录会有一次性验证步骤）
3. 登录后进入首页，点击右上角或页面中央的 **「创建 Agent」** 按钮

> 提示：首次登录会看到一个简短的新手引导弹窗，点击「开始引导」可以跟着提示一步步走。如果不需要，点击右上角「×」跳过即可。

---

### Step 2：从模板库选择「会议纪要助手」

1. 点击「创建 Agent」后进入模板选择页面
2. 在模板库中找到 **「会议纪要助手」**（在「通用类」分类下）
3. 点击模板卡片，查看模板描述
4. 点击 **「使用此模板」**

> 提示：如果一时找不到，可以在模板库搜索框输入「会议」，快速定位。

---

### Step 3：填写 Agent 基本信息（30 秒完成）

进入 Agent 配置页面后，只需填写两个字段：

- **Agent 名称**：建议填「[你的姓名] 会议纪要」，例如「张三 会议纪要」，方便日后区分
- **工作语言**：选「中文」

其余选项保持默认，直接点击 **「保存并开始对话」**。

---

### Step 4：发送第一条消息，看看 Agent 怎么工作

Agent 创建成功后，会自动进入对话界面。把下面这段测试内容复制，粘贴进对话框，点击发送：

```
参会人：张三、李四、王五
会议时间：今天上午 10:00
讨论内容：
1. Q2 营销预算，决定增加 20%，由李四负责执行方案，下周五前提交
2. 新官网上线时间定为下月 15 日，王五负责对接技术部
3. 下次会议时间：下周四上午 10:00
```

**预期输出**：Agent 会在 10 秒内生成一份格式规范的会议纪要，包含：
- 会议基本信息（时间、参会人）
- 决议事项列表
- 责任人和截止时间
- 下次会议安排

---

### Step 5：尝试追问——这才是 Agent 真正强大的地方

会议纪要生成后，在同一对话框继续输入：

> 「把行动项单独整理成一个表格，列出：任务、负责人、截止日期」

Agent 会立即在对话中补充这个表格，不需要重新创建 Agent，也不需要重新粘贴原始记录。

> 关键认知：Agent 是持续对话的工作伙伴，不是一次性问答工具。你可以不断追问、调整，直到输出满足你的需求。

### 5 分钟快速上手总结

```
打开平台 → 点击「创建 Agent」→ 选模板 → 填名称 → 开始对话
```

创建完成后，这个 Agent 会一直保存在你的账户下。下次打开「我的 Agents」列表，点击它就能继续上次的工作，历史记录完整保留。

## 三、Wave 3 新功能：Agent Teams 多 Agent 协作

### 13:00–20:00

---

### 什么是 Agent Teams？

Agent Teams 是 Wave 3 新增的功能。简单来说：

> **Agent Teams = 让多个 Agent 像一条流水线一样协同工作**

之前，你需要手动把一个 Agent 的输出复制给另一个 Agent。现在，你可以创建一个 Team 对话，在一次对话中让多个 Agent 自动接力完成任务。

**一个直观的例子**：

不用 Teams：
```
你 → 代码审查 Agent（获得审查结果）→ 手动复制 → 文档撰写 Agent（生成文档）
```

用 Teams：
```
你发一条消息 → 代码审查 Agent 自动审查 → 文档撰写 Agent 自动接收结果并生成文档
```

---

### Agent Teams 的三大典型场景

#### 场景一：内容创作流水线（适合运营、产品、HR）

**成员配置**：写作助手 + 文档助手（审校角色）+ 邮件撰写助手

**工作流程**：
1. 你说：「帮我写一篇关于 Q2 产品更新的内部公告，受众是全体员工」
2. 写作助手生成初稿
3. 文档助手自动对初稿进行审校，优化措辞和结构
4. 邮件撰写助手将审校后的内容格式化为可直接发送的邮件格式

**你需要做的**：只是在开头说一句话。

---

#### 场景二：数据分析报告（适合财务、运营、产品）

**成员配置**：财务数据分析助手 + 文档助手

**工作流程**：
1. 你粘贴原始数据和分析需求
2. 财务数据分析助手完成数据解读和关键指标提炼
3. 文档助手接收分析结果，自动生成格式规范的报告框架

**省去的步骤**：不再需要手动将数据分析结果整理成报告格式。

---

#### 场景三：代码审查 + 文档自动生成（适合技术团队）

**成员配置**：代码审查助手 + 技术方案助手

**工作流程**：
1. 你粘贴代码片段
2. 代码审查助手给出审查意见和修改建议
3. 技术方案助手根据审查结果自动生成该模块的接口文档或变更说明

---

### 如何创建一个 Agent Team？

**前提**：你至少已经创建了两个 Agent（参考第二章快速开始）。

**步骤**：

1. 在平台首页，点击 **「创建 Team 对话」**（与「创建 Agent」按钮相邻）
2. 输入 Team 名称（如「内容创作流水线」）
3. 从你的 Agent 列表中选择参与的 Agent（可多选，建议先从 2 个开始）
4. 点击 **「开始 Team 对话」**
5. 在对话框中发送你的任务描述，Team 中的 Agent 会按顺序接力处理

> 新手建议：第一次使用 Teams，先选 2 个 Agent 体验，熟悉协作模式后再加入更多 Agent。

---

### Agent Teams 使用注意事项

| 注意点 | 说明 |
|-------|------|
| Agent 顺序 | Team 中 Agent 的处理顺序在创建时可以调整，先后顺序影响工作流效果 |
| 适用场景 | Teams 最适合有明确先后依赖关系的任务，随意组合不一定有效 |
| 输出检查 | 每个 Agent 的输出仍建议人工确认，尤其是最终输出 |
| 从单 Agent 开始 | 如果不确定是否需要 Teams，先用单个 Agent 完成任务，再考虑是否引入协作 |

### 技术团队

**推荐模板**：代码审查助手 / 技术方案助手

#### 代码审查助手

**什么时候用**：提交前的代码自查，或快速审查他人代码。

1. 创建 Agent，选择「代码审查助手」模板
2. 在配置中选择主要编程语言（Python / JavaScript / Go / 其他）
3. 将需要审查的代码片段粘贴进对话框，说明审查重点：

   > 「检查这段代码是否有安全漏洞，以及是否符合 PEP8 规范」

4. 查看审查意见，可追问：

   > 「针对第 3 条给出修改后的代码示例」

**实际效果**：平均单次代码审查从 30 分钟缩短到 8 分钟（Wave 2 用户数据）。

---

#### 技术方案助手

**什么时候用**：需要快速生成技术方案初稿或梳理模块设计思路。

1. 选择「技术方案助手」模板，创建 Agent
2. 描述你要解决的技术问题和约束条件：

   > 「需要设计一个支持 10 万 QPS 的消息推送服务，主要约束是延迟要低于 100ms，现有技术栈是 Python + Redis」

3. 获取方案框架，可追问：「把第二方案的架构图用文字描述出来」「加上监控和告警的设计」

## 五、常见问题解答

### 27:00–29:30

---

**Q1：我完全不懂 AI 技术，能用这个平台吗？**

完全可以。平台设计的核心目标就是"零技术背景也能上手"。具体来说：

- 不需要写任何代码或技术配置
- 和 Agent 对话就像在微信上发消息，用自然语言说你想要什么即可
- 模板库已经预设好了各种工作场景，直接选择使用
- 不满意可以直接在对话中说「改成……」，Agent 会重新生成

Wave 1 中满意度最高的用户之一，就是完全没有技术背景的运营同学。

---

**Q2：我的数据安全吗？聊天内容会被外部看到吗？**

不会。关键信息：

- 平台部署在公司自有服务器，对话数据不会传输到任何外部公司（包括 OpenAI、Anthropic 等）
- 对话记录仅你本人和平台管理员可访问
- 受公司数据安全策略保护

建议：如需输入含有敏感数字（如具体营收金额）或客户姓名的内容，可以用「某客户」「XX 万元」等方式脱敏处理后再输入，这是一个良好的数据使用习惯。

---

**Q3：Agent 说错了或者输出质量不好，怎么办？**

有三个快速解决方法：

1. **直接纠正**：在对话中说「这个不对，请按照……修改」，Agent 会立即调整
2. **补充信息**：输入的背景越详细，输出质量越高。比如「面向管理层」和「面向普通员工」会让文档风格完全不同
3. **换个说法**：换一种表达方式描述你的需求，往往能得到更好的结果

如果某类任务持续输出不符合预期，可以通过平台内的「反馈」按钮提交，帮助持续改进模板。

---

**Q4：Agent Teams 和单个 Agent 有什么区别？我应该用哪个？**

新手建议：先用单个 Agent，熟悉平台后再尝试 Teams。

使用单个 Agent 的场景：
- 单一类型的任务（只需要写一封邮件、只需要审查一段代码）
- 任务之间没有明确的先后依赖关系

使用 Agent Teams 的场景：
- 任务有清晰的流水线结构（A 完成后 B 才能开始）
- 你需要不同专业角色的 Agent 依次处理同一个输入（例如：分析→写作→格式化）

---

**Q5：一个 Agent 创建一次就够了吗？还是每次都要重新创建？**

创建一次，永久保存。进入「我的 Agents」列表，点击对应 Agent 即可继续上次的对话，历史记录完整保留。

建议：为每个常用工作场景各保留一个专属 Agent，不要用同一个 Agent 混合处理太多不同类型的任务，专注的 Agent 输出质量更稳定。

---

**Q6：可以把我的 Agent 分享给团队同事吗？**

可以。在 Agent 设置页面找到「分享」选项，可以：
- 生成分享链接发给同事
- 直接邀请指定同事使用

分享后，同事可以用你的 Agent 进行对话，但无法修改 Agent 的配置和角色设定。这对于团队内推广"标准化工作流"非常有用。
