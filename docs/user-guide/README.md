## 平台简介

AI Agents Platform 是基于 Amazon Bedrock AgentCore 的企业内部 AI Agents 平台。平台帮助企业团队创建、部署和管理 AI Agent，通过自然语言对话完成业务任务。

**核心价值**:

- **降低门槛**: 非技术人员也能通过模板创建 AI Agent
- **统一管理**: 集中管理所有 Agent、工具和知识库
- **安全合规**: 完整的 RBAC 权限控制和审计追踪
- **成本可控**: Token 消耗归因和 AWS 成本可视化

## 角色与权限

平台采用基于角色的访问控制 (RBAC)，定义了三种角色:

### ADMIN (管理员)

- 管理所有用户账户（启用/停用、角色分配）
- 审批/拒绝工具注册申请
- 查看全局使用洞察和审计日志
- 操作所有 Agent（不限于自己创建的）
- 导出审计日志 CSV

### DEVELOPER (开发者)

- 创建和管理自己的 Agent
- 注册工具并提交审批
- 创建和管理知识库
- 发布 Agent 模板
- 创建测试集和运行评估
- 查看自己的使用数据

### VIEWER (查看者)

- 查看已发布的模板
- 与已激活的 Agent 对话
- 查看自己的使用数据
- 浏览已批准的工具列表

### 权限对照表

| 操作 | ADMIN | DEVELOPER | VIEWER |
|------|:-----:|:---------:|:------:|
| 创建 Agent | Y | Y | - |
| 编辑/删除 Agent | 所有 | 自己的 | - |
| 与 Agent 对话 | Y | Y | Y |
| 注册工具 | Y | Y | - |
| 审批工具 | Y | - | - |
| 创建知识库 | Y | Y | - |
| 发布模板 | Y | Y | - |
| 查看审计日志 | Y | - | - |
| 查看全局洞察 | Y | - | - |

## 常见问题

### Q1: 我的账号如何获取？

联系平台管理员 (ADMIN) 为你创建账号。如果开启了自助注册，可以通过注册页面自行注册。注册后默认角色为 VIEWER，如需更高权限请联系管理员调整。

### Q2: VIEWER 角色能做什么？

VIEWER 可以与已激活的 Agent 进行对话、浏览已发布的模板、查看已批准的工具列表，以及查看自己的使用数据。如果需要创建 Agent 或管理资源，请联系管理员升级为 DEVELOPER 角色。

### Q3: 平台支持哪些 AI 模型？

平台基于 Amazon Bedrock，支持 Claude 系列模型:

| 模型 | 适用场景 | 成本 |
|------|---------|------|
| Claude Haiku | 简单对话、快速响应（默认） | 低 |
| Claude Sonnet | 代码生成、复杂分析 | 中 |
| Claude Opus | 深度推理、高质量输出 | 高 |

### Q4: 对话消息有长度限制吗？

单条消息最大 2000 字符。对话历史采用滑动窗口机制，超出上下文窗口的早期消息会被自动截断以确保 LLM 处理效率。

### Q5: 如何报告平台问题？

请联系平台管理员，提供以下信息:

1. 问题发生的时间
2. 你执行的操作步骤
3. 看到的错误信息（如有）
4. 浏览器和操作系统版本
