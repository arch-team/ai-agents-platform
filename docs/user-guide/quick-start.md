## 前置条件

- 已获取平台账号（DEVELOPER 或 ADMIN 角色）
- 使用 Chrome 90+ 或其他支持的现代浏览器
- 可访问平台 URL（联系管理员获取地址）

---

## 步骤一: 登录平台

1. 在浏览器中打开平台地址
2. 在登录页面输入邮箱和密码
3. 点击 **登录** 按钮
4. 登录成功后自动跳转到控制台首页

> [登录页面: 包含邮箱输入框、密码输入框和登录按钮]

**API 说明**: 登录接口为 `POST /api/v1/auth/login`，返回 JWT Access Token 和 Refresh Token。Access Token 有效期为 30 分钟，过期后使用 Refresh Token 自动续期。

---

## 步骤二: 创建 Agent

1. 在左侧导航栏点击 **Agent 管理**
2. 点击页面右上角的 **创建 Agent** 按钮
3. 填写基本信息:

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | Agent 的名称，同一用户下不可重复 | "客服助手" |
| 描述 | Agent 的用途描述（可选） | "处理客户常见问题咨询" |

4. 点击 **创建** 完成

> [创建 Agent 表单: 包含名称、描述输入框和创建按钮]

创建成功后，Agent 处于 **DRAFT（草稿）** 状态，还不能用于对话。

---

## 步骤三: 配置 System Prompt

System Prompt 是 Agent 的核心配置，决定了 Agent 的行为模式和回答风格。

1. 在 Agent 列表中点击刚创建的 Agent 进入详情页
2. 在 **System Prompt** 编辑区域输入提示词

**示例 System Prompt**:

```
你是一个专业的客服助手，负责解答客户关于产品使用的问题。

规则:
1. 使用友好、专业的语气
2. 如果不确定答案，请诚实告知并建议联系人工客服
3. 回答应简洁明了，控制在 200 字以内
4. 涉及账户安全问题时，引导用户联系安全团队
```

3. 根据需要调整模型参数:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| model_id | us.anthropic.claude-3-5-haiku-20241022-v1:0 | AI 模型选择 |
| temperature | 1.0 | 创造性程度 (0.0-1.0，越高越有创造性) |
| max_tokens | 2048 | 单次回复最大 Token 数 |
| top_p | 0.999 | 采样概率阈值 |

4. 点击 **保存** 应用配置

> [Agent 详情页: 包含 System Prompt 编辑器、模型参数配置面板和保存按钮]

---

## 步骤四: 激活 Agent

Agent 必须激活后才能用于对话。

1. 在 Agent 详情页点击 **激活** 按钮
2. 系统验证配置完整性（名称和 System Prompt 不能为空）
3. Agent 状态变为 **ACTIVE（已激活）**

> [Agent 状态变化: DRAFT -> 点击激活 -> ACTIVE]

**注意**: 激活后仍可编辑 Agent 配置。如果需要下线 Agent，使用 **归档** 操作（归档不可逆）。

---

## 步骤五: 发起对话

1. 在左侧导航栏点击 **对话** 或在 Agent 详情页点击 **发起对话**
2. 选择要对话的 Agent（需为 ACTIVE 状态）
3. 在消息输入框输入你的问题
4. 点击 **发送** 或按回车键

> [对话界面: 左侧为对话列表，右侧为消息区域和输入框，Agent 回复以流式方式逐字显示]

Agent 的回复以 **SSE 流式** 方式实时显示，你可以看到文字逐步生成。

### 对话操作

| 操作 | 说明 |
|------|------|
| 发送消息 | 在输入框输入文本并发送 |
| 查看历史 | 在对话列表中选择历史对话 |
| 结束对话 | 点击对话右上角的 **结束** 按钮 |

## 常见问题

### Q1: 创建 Agent 时提示"名称已存在"？

同一用户下 Agent 名称不能重复。请更换一个不同的名称，或者检查是否已有同名的草稿 Agent。

### Q2: 激活按钮点击后提示失败？

激活需要满足以下条件:

- Agent 名称不能为空
- System Prompt 不能为空
- Agent 当前状态必须为 DRAFT

请检查 System Prompt 是否已填写。

### Q3: 对话没有回复或回复很慢？

可能的原因:

1. **网络问题**: 检查与平台服务器的网络连接
2. **模型负载**: 高峰期模型响应可能较慢，请稍后重试
3. **Token 超限**: 如果对话历史过长，尝试结束当前对话并开始新对话

### Q4: 如何使用 Agent 预览功能？

在 Agent 详情页的 **测试面板** 中，可以直接输入提示语测试 Agent 回复效果，无需创建正式对话。这对调试 System Prompt 非常有用。
