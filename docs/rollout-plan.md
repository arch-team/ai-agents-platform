# 梯度推广计划

> AI Agents Platform 全公司推广方案 — 10 → 30 → 50 用户

---

## 1. 推广目标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 活跃用户数 | >= 50 人 | 最近 7 天有登录和对话的用户 |
| 非技术自助创建比例 | >= 40% | 非 DEVELOPER 角色创建的 Agent 占比 |
| 用户满意度 | >= 4.0/5.0 | 推广期间收集的反馈评分 |
| 平台可用性 | >= 99.9% | API 可用性监控 |

---

## 2. 三阶段推广策略

### Wave 1: 种子用户 (10 人) — Week 4

**目标**: 核心验证，收集深度反馈

| 维度 | 计划 |
|------|------|
| **用户画像** | 5 名技术人员 (DEVELOPER) + 3 名产品/运营 (VIEWER→DEVELOPER) + 2 名管理者 (ADMIN) |
| **选择标准** | 对 AI 有兴趣 + 愿意提供反馈 + 来自不同部门 |
| **入职方式** | 1-on-1 Demo + 手把手创建第一个 Agent |
| **支持等级** | 专属微信群/Slack 频道，24h 响应 |
| **反馈机制** | 每日 standup 收集问题 + 周末汇总会 |
| **退出标准** | 10 人全部完成 Onboarding + 各自创建至少 1 个 Agent |

**Wave 1 Checklist**:
- [ ] 创建 10 个用户账户，分配角色
- [ ] 准备 Demo 脚本和 PPT
- [ ] 创建支持沟通群
- [ ] 准备 3 个预置 Agent 模板（文档助手/代码审查/会议纪要）
- [ ] 设置反馈收集表单

### Wave 2: 扩展推广 (30 人) — Week 5

**目标**: 规模验证，测试自助能力

| 维度 | 计划 |
|------|------|
| **用户画像** | 技术团队 + 产品 + 运营 + HR + 财务 |
| **选择标准** | 部门代表 + Wave 1 用户推荐 |
| **入职方式** | 集体培训会 (30min) + 自助文档 + Wave 1 用户作为内部导师 |
| **支持等级** | 公共 FAQ + Wave 1 导师 + 每周答疑 |
| **反馈机制** | 周度问卷调查 + Issue 模板 |
| **退出标准** | 30 人完成 Onboarding + >= 10 个 Agent 被创建 |

**Wave 2 Checklist**:
- [ ] 批量创建 20 个新用户账户
- [ ] 举办 30 分钟培训会（录屏备用）
- [ ] Wave 1 用户中选出 3 名内部导师
- [ ] 整理 Wave 1 反馈并修复高优问题
- [ ] 发布 5 个新预置模板（覆盖非技术场景）

### Wave 3: 全公司推广 (50+ 人) — Week 6

**目标**: 达成 50 活跃用户 + 40% 自助创建

| 维度 | 计划 |
|------|------|
| **用户画像** | 全公司所有感兴趣的员工 |
| **入职方式** | 自助注册 + 文档 + FAQ + 培训录屏 |
| **支持等级** | 自助为主 + 公共答疑频道 + 每双周 Office Hour |
| **反馈机制** | 持续 Issue 模板 + 季度满意度调查 |
| **成功标准** | >= 50 活跃用户 + >= 40% 非技术自助创建 |

**Wave 3 Checklist**:
- [ ] 开放自助注册（或批量邀请）
- [ ] 发布培训录屏到内部知识库
- [ ] 整理并发布完整 FAQ (20+ 问题)
- [ ] 设置自动化 Onboarding 邮件流程
- [ ] 配置使用洞察仪表板供管理层查看

## 3. 反馈修复流程

### 问题分级

| 级别 | 定义 | 响应时间 | 修复时间 |
|------|------|---------|---------|
| P0 | 平台不可用、数据丢失 | 1 小时 | 4 小时 |
| P1 | 核心功能异常（对话失败、创建失败） | 4 小时 | 1 个工作日 |
| P2 | 体验问题（UI 不友好、性能慢） | 1 个工作日 | 1 周 |
| P3 | 功能建议、非阻塞问题 | 1 周 | 下个迭代 |

### 反馈收集渠道

1. **GitHub Issues** (推荐): 使用 Bug 报告 / 功能建议模板
2. **支持频道**: 微信群/Slack（Wave 1-2 专属）
3. **反馈表单**: Google Form / 飞书表单
4. **Office Hour**: 每双周 30 分钟在线答疑

---

## 4. 风险缓解

| 风险 | 缓解策略 |
|------|---------|
| 用户增长不达预期 | 邀请制 + 内部导师 + 部门推荐 + 管理层支持 |
| 非技术用户学习成本高 | 预置模板 + 简化创建流程 + 视频教程 |
| 平台性能瓶颈 | 压测已验证 50 并发；ECS 自动扩展 |
| 安全合规顾虑 | 审计日志全覆盖 + RBAC 权限控制 |
| 模型成本超预期 | 默认 Haiku + Token 消耗监控 + 使用洞察仪表板 |
