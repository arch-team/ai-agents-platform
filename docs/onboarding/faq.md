# 常见问题解答 (FAQ)

> AI Agents Platform 使用过程中的常见问题

---

## 账户与权限

### Q1: 平台有哪些角色？各有什么权限？

| 角色 | 权限范围 |
|------|---------|
| **ADMIN** | 全部操作：用户管理、工具审批、审计日志查看、所有 Agent 管理 |
| **DEVELOPER** | 创建和管理自己的 Agent、注册工具、创建知识库、使用模板 |
| **VIEWER** | 只读访问：查看 Agent 列表、浏览模板、查看公开的使用洞察 |

### Q2: 如何申请更高权限？

联系平台管理员 (ADMIN)，说明需要的角色和用途。管理员可在用户管理页面调整角色。

### Q3: 忘记密码怎么办？

联系管理员重置密码。当前版本暂不支持自助密码重置功能。

## Agent 使用

### Q4: Agent 创建后为什么不能直接对话？

新创建的 Agent 处于 `DRAFT` 状态，需要先点击 **激活** 按钮变为 `ACTIVE` 状态才能开始对话。激活前请确认 System Prompt 和配置参数已正确设置。

### Q5: 如何选择合适的 AI 模型？

| 模型 | 适用场景 | 特点 |
|------|---------|------|
| Claude 3.5 Haiku | 简单问答、信息提取、分类 | 速度最快、成本最低 |
| Claude Sonnet 4 | 代码生成、技术分析、文档撰写 | 编码能力强、性价比高 |
| Claude Opus 4.6 | 复杂推理、长文档分析、多步决策 | 最强推理、1M 上下文窗口 |

**建议**: 从 Haiku 开始，如果效果不满意再升级模型。

### Q6: temperature 参数如何设置？

- **0.0 - 0.3**: 高确定性，适合事实性问答、代码生成
- **0.4 - 0.7**: 平衡创造性和准确性（推荐默认值）
- **0.8 - 1.0**: 高创造性，适合头脑风暴、创意写作

### Q7: Agent Teams 是什么？

Agent Teams 允许一个 Agent 自主组建团队来完成复杂任务。启用后，Agent 可以：
- 创建多个 Teammate
- 分配和追踪任务
- Teammate 间协作沟通

**注意**: Agent Teams 消耗 Token 较多（~800K tokens/次），建议仅用于确实需要多角色协作的复杂任务。

### Q8: 如何编写好的 System Prompt？

**基本结构**:
```
你是 [角色描述]。

## 核心职责
- [职责 1]
- [职责 2]

## 行为规范
- [规范 1]
- [规范 2]

## 输出格式
[期望的输出格式说明]
```

**最佳实践**:
1. 角色定义清晰
2. 职责范围明确
3. 输出格式规范化
4. 避免过于冗长（建议 500-2000 字符）

## 工具集成

### Q9: 支持哪些类型的工具？

1. **MCP Server**: 遵循 MCP 协议的服务端，支持 stdio/SSE 传输
2. **API**: REST API 端点，支持 GET/POST/PUT/DELETE
3. **Function**: 代码函数，运行在沙盒环境中

### Q10: 工具审批需要多长时间？

工具提交后进入 `PENDING_REVIEW` 状态，由 ADMIN 审批。通常在 1-2 个工作日内完成审批。

### Q11: 审批被拒绝后怎么办？

查看审批评论了解拒绝原因，修改后通过 **重新提交** 按钮提交审批。

## 知识库

### Q12: 支持哪些文档格式？

当前支持以下格式：
- PDF 文档
- TXT 纯文本
- Markdown (.md) 文件

### Q13: 文档上传后多久可以使用？

文档上传后需要经过处理和索引（`UPLOADING → PROCESSING → INDEXED`），通常在几分钟内完成。大型文档可能需要更长时间。

### Q14: 知识库检索准确吗？

知识库使用 Amazon Bedrock Knowledge Bases 进行语义检索（RAG），Top-5 召回率 >= 80%。检索结果会自动作为上下文注入到 Agent 的对话中。

## 性能与成本

### Q15: 对话响应速度如何？

- **非 LLM 接口** (创建/查询/列表): P95 < 300ms
- **LLM 对话**: 首 Token 延迟约 1-3 秒，之后通过 SSE 流式逐步输出

### Q16: 如何控制 Token 消耗？

1. 选择合适的模型（Haiku 成本最低）
2. 控制 `max_tokens` 参数
3. 合理使用对话历史（系统自动进行滑动窗口截取）
4. 避免不必要的 Agent Teams 使用

### Q17: 在哪里查看使用情况？

进入 **使用洞察** 页面，可查看：
- Token 消耗统计（按 Agent / 时间段）
- 成本归因（通过 AWS Cost Explorer 获取真实账单）
- 使用趋势图

## 故障排除

### Q18: 对话时出现错误怎么办？

1. 检查 Agent 是否处于 ACTIVE 状态
2. 确认网络连接正常
3. 如果使用 Agent Teams，检查是否超时（默认 30 分钟）
4. 联系管理员查看审计日志

### Q19: SSE 流式响应中断怎么办？

SSE 连接可能因网络波动中断。刷新页面后可在对话历史中查看已完成的部分，然后继续对话。

### Q20: 如何报告 Bug？

使用 [Bug 报告模板](README.md) 提交 Issue，包含：
- 问题描述
- 复现步骤
- 期望行为 vs 实际行为
- 浏览器和操作系统信息
