## 一、培训大纲

### 00:00–05:00 开场：AI Agents 能解决哪些工作痛点？

**AI Agent 是什么？**

AI Agent 是一个"会记住上下文、可以执行专项任务"的智能助手。与普通 AI 聊天不同，Agent 有明确的角色设定，每次对话都在同一个工作场景下持续运作，不需要反复解释背景。

**3 个真实案例（来自 Wave 1 用户）**

| 场景 | 以前怎么做 | 用 Agent 之后 |
|------|-----------|-------------|
| 会议纪要整理 | 录音转文字→手动整理→写邮件，约 45 分钟 | 粘贴原始记录→Agent 输出结构化纪要，5 分钟内完成 |
| 代码审查 | 逐行检查，容易遗漏规范问题 | 提交代码片段→Agent 按规范给出审查意见和改进建议 |
| 邮件撰写 | 反复措辞、确认语气是否合适 | 描述意图→Agent 生成初稿，微调后发出 |

**核心价值**：把重复性、格式化的脑力劳动交给 Agent，把时间留给真正需要判断的工作。

### 05:00–15:00 实操演示：以「会议纪要助手」为例

> 所有部门都开会，这个场景最容易上手。

**完整流程演示（10 分钟）**

**Step 1：进入平台，选择模板**

1. 打开平台（地址见本文第五部分）
2. 点击首页「创建 Agent」按钮
3. 在模板库中选择「会议纪要助手」
4. 点击「使用此模板」

**Step 2：完成 Agent 配置**

- Agent 名称：建议填写「[你的姓名] 会议纪要助手」，方便日后区分
- 工作语言：选择中文
- 其余设置保持默认即可
- 点击「保存并开始对话」

**Step 3：发起第一次对话**

将以下测试内容粘贴进对话框，点击发送：

```
参会人：张三、李四、王五
会议时间：今天上午 10:00
讨论内容：
1. Q2 营销预算，决定增加 20%，由李四负责执行方案
2. 新官网上线时间定为下月 15 日，王五对接技术部
3. 下次会议时间：下周四 10:00
```

**预期输出**：Agent 会输出格式规范的会议纪要，包含会议信息、决议事项、责任人、跟进时间。

**演示要点**：
- 展示 Agent 输出后，可以追问「把行动项单独列一个表格」——Agent 会继续在同一对话中完善
- 展示「历史对话」功能：下次开会时打开同一个 Agent，上下文会继续保留

## 二、分部门实操指引

### 技术团队

**推荐模板**：代码审查助手 / Python 助手 / SQL 助手

**代码审查助手（3 步）**

1. 创建 Agent，选择「代码审查助手」模板，语言选 Python/JavaScript/其他
2. 将需要审查的代码片段粘贴进对话框，描述审查重点（如：「检查是否有潜在的安全漏洞」）
3. 查看审查建议，可追问「针对第 2 条给出修改示例」

**Python 助手（4 步）**

1. 选择「Python 助手」模板，创建 Agent
2. 描述你要实现的功能（如：「写一个读取 Excel 并计算列均值的脚本」）
3. 查看生成的代码，直接复制使用或追问修改
4. 如需调试，粘贴报错信息，Agent 会帮你分析原因

**SQL 助手（3 步）**

1. 选择「SQL 助手」模板，创建 Agent
2. 描述你的表结构和查询需求（如：「orders 表有 id, user_id, amount, created_at，查询每个用户过去 30 天的消费总额」）
3. 获取 SQL 语句，可追问「加上分页」「用 CTE 重写」

---

### 产品 / 运营 / HR

**推荐模板**：文档助手 / 会议纪要助手 / 邮件撰写助手

**文档助手（5 步）**

1. 选择「文档助手」模板，创建 Agent
2. 说明文档类型（如：产品需求文档 / 活动方案 / 员工手册章节）
3. 提供核心信息（要点、背景、受众）
4. 查看 Agent 生成的初稿
5. 追问调整：「把第二部分改得更简洁」「加一个风险说明章节」

**会议纪要助手**：参考培训演示部分，步骤完全相同。

**邮件撰写助手（4 步）**

1. 选择「邮件撰写助手」模板，创建 Agent
2. 描述邮件目的、收件人、核心内容（如：「给供应商发邮件，礼貌催促对方本周内提供报价单」）
3. 查看生成的邮件草稿
4. 根据实际情况微调主题行和称呼后发出

### 财务

**推荐模板**：财务报告助手

**财务报告助手（5 步）**

1. 选择「财务报告助手」模板，创建 Agent
2. 输入报告期间和核心数据（如：月度/季度收入、支出、利润数字）
3. 说明报告受众（如：向管理层汇报 / 内部月报）
4. 查看 Agent 生成的报告框架和分析描述
5. 追问优化：「添加同比增长率的分析」「把结论段改为更直接的管理层摘要风格」

**注意**：Agent 辅助文字分析和格式化，数据准确性需自行核实。

### 法务

**推荐模板**：法务文档助手

**法务文档助手（4 步）**

1. 选择「法务文档助手」模板，创建 Agent
2. 描述文档类型和关键条款需求（如：「起草一份软件服务协议框架，包含服务范围、保密条款、违约责任」）
3. 查看生成的框架文本
4. 追问补充：「细化第三条违约责任的赔偿计算方式」

**重要提示**：Agent 生成的法律文本为参考框架，正式文件需经法务专业人员审核确认。

### 销售 / 客服

**推荐模板**：销售话术助手 / 智能客服助手

**销售话术助手（4 步）**

1. 选择「销售话术助手」模板，创建 Agent
2. 描述产品或服务、目标客户类型、对话场景（如：「客户首次询价时的开场白」）
3. 查看 Agent 提供的话术示例
4. 追问优化：「客户说价格太高，给我一个应对话术」「改成更轻松的对话风格」

**智能客服助手（5 步）**

3. 测试：输入一个典型客户问题
4. 查看回答是否准确，不准确处追问调整
5. 使用稳定后，可将对话记录分享给团队参考

## 四、常见问题 FAQ

**Q1：Agent 和普通 AI 聊天（如 ChatGPT）有什么区别？**

普通 AI 聊天每次对话都是全新开始，没有记忆，也没有角色设定。我们平台的 Agent 有三个核心优势：
- **角色固定**：每个 Agent 有专属的工作定位，不需要每次重新介绍背景
- **上下文持续**：同一个 Agent 的对话历史会保留，下次打开继续工作
- **部署在企业内部**：数据不会发送到外部消费级服务（见 Q3）

---

**Q2：我不懂 AI，能用这个平台吗？**

可以。平台设计的出发点就是"非技术用户也能用"。具体来说：
- 首次使用有步骤引导教程，按提示点击即可
- 模板库已预设好各种工作场景，不需要自己写任何技术配置
- 和 Agent 对话就像发消息，用自然语言描述你想要什么即可

如果遇到困惑，可以联系非技术导师运营人员01（见第三部分）。

---

**Q3：数据安全吗？我的工作内容会被保存吗？**

- 平台部署在公司内部，对话数据不会发送到任何外部公司
- 对话记录存储在公司服务器，受公司数据安全策略保护
- 仅你本人和平台管理员可访问你的对话记录
- 如有更高保密级别的内容，建议脱敏处理后再输入（如替换具体金额、客户名称）

---

**Q4：每次对话都要重新创建 Agent 吗？**

不需要。Agent 创建一次后会一直保存在你的账户下。进入「我的 Agents」列表，点击对应 Agent 即可继续上次的对话，历史记录完整保留。建议为每个常用工作场景各创建一个 Agent。

---

**Q5：可以把我的 Agent 分享给同事吗？**

可以。在 Agent 设置页面，有「分享」选项，可以生成分享链接或直接邀请指定同事使用。分享后，同事可以用你的 Agent 对话，但无法修改 Agent 的配置。

---

**Q6：Agent 回答的内容不准确，怎么办？**

有几种方法：
1. **追问纠正**：直接在对话中说「这个不对，请修改……」，Agent 会根据你的反馈调整
2. **提供更多背景**：输入的信息越具体，输出质量越高
3. **换个提问方式**：有时换一种表达方式可以得到更好的结果
4. 如果发现某类问题 Agent 持续回答不准，可以通过反馈入口报告，帮助改进

---

**Q7：一个 Agent 可以处理多种工作吗，还是每种工作要建一个？**

建议按工作场景分开建。虽然一个 Agent 理论上可以处理多种任务，但有独立角色设定的 Agent 输出质量更稳定。例如：「会议纪要助手」专注纪要整理，「邮件助手」专注邮件撰写，不混用效果更好。

---

**Q8：输入框有字数限制吗？可以粘贴很长的文档吗？**

单次输入有长度限制，具体限制见平台帮助文档。对于较长的文档，建议：
- 分段粘贴，每次处理一个章节
- 只粘贴关键内容，而非整份文档
- 告诉 Agent 你只需要处理文档的某个部分

---

**Q9：平台会更新吗？新模板怎么获取？**

平台会持续更新，包括新增模板、功能优化等。更新通知会通过以下渠道发布：
- 平台内公告（登录后首页可见）
- Wave 用户群通知

新模板上线后，在「模板库」中直接可见，无需额外操作。

---

**Q10：如果我发现了 bug 或者有功能建议，怎么反馈？**

- **平台内反馈**：每个对话页面右上角有「反馈」按钮，可提交问题描述和截图
- **直接联系导师**：Wave 1 导师可以帮助判断是操作问题还是平台问题
- **每周答疑**：在固定答疑时间提出（见第五部分）

你的反馈对平台改进非常重要，欢迎积极提交。
